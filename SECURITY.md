# Security policy

Please report suspected vulnerabilities privately so they can be understood
and fixed before details are shared publicly.

## Supported versions

Security fixes target the latest released version of each Mateo package and the
current design-system release. Older package versions may receive a fix when
the impact and compatibility risk justify it.

## Report a vulnerability

Use a
[private GitHub security advisory](https://github.com/Ventairy/mateo/security/advisories/new)
or email `security@ventairy.com`. Include:

- the affected package, design artifact, and version or commit;
- a clear description of the issue and its potential impact;
- reproduction steps or a minimal example when possible;
- any known mitigations or disclosure constraints.

Do not open a public issue for a suspected vulnerability. Do not include secrets
or personal data that are not necessary to reproduce the report.

## What to expect

Maintainers aim to acknowledge a report within three business days and provide
an initial assessment within seven business days. Complex reports may take
longer. Valid fixes are prepared privately, released through the affected
package lane, and disclosed in coordination with the reporter when practical.
